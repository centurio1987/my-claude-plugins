# 질문 검토 스킬

면접 질문과 사용자의 답변을 받아 세 가지 기준으로 평가하고, 구체적인 개선 피드백을 한국어로 제공한다.

## 실행 순서

### 1단계: 입력 수집

다음을 확인한다 (이미 대화에 있으면 별도 요청 불필요):

- **질문**: 평가 대상 면접 질문
- **답변**: 사용자의 답변
- **포지션** (선택): 포지션을 알면 맥락에 맞게 평가 기준 조정 가능

### 2단계: 3가지 기준으로 평가

각 기준을 1~5점으로 채점하고 피드백을 작성한다.

#### 기준 1: 의도 부합도 (1~5점)
질문이 묻는 핵심을 답변이 얼마나 정확히 커버하는가.

- **5점**: 질문의 모든 핵심 포인트를 명확히 다루고, 불필요한 이탈 없음
- **3점**: 핵심은 다루나 일부 빠진 포인트 있음, 또는 관련성 낮은 내용 포함
- **1점**: 질문의 의도와 다른 방향으로 답변

평가 시 확인할 항목:
- 질문에서 요구한 개념/경험을 직접 언급했는가
- 답변 방향이 질문의 맥락(기술적, 경험적, 행동 기반)과 일치하는가
- 불필요한 서두나 주제 이탈이 없는가

#### 기준 2: 깊이 (1~5점)
답변이 충분히 심층적인가.

- **5점**: 개념 설명 + 실무 경험 + 트레이드오프/근거 + 수치/결과까지 포함
- **3점**: 개념은 설명하나 경험이나 근거가 얕음
- **1점**: 개념 정의만 나열, 실질적 이해 없음

평가 시 확인할 항목:
- 단순 정의 나열인가, 실제 경험에 기반한 설명인가
- 왜 그 방법을 선택했는지 근거를 제시하는가
- 트레이드오프나 대안을 인식하고 있는가
- 구체적인 수치, 상황, 결과가 포함되어 있는가

#### 기준 3: 톤앤매너 (1~5점)
인터뷰에 적절한 언어와 태도인가.

- **5점**: 전문적이고 명확하며 자신감 있는 표현, 지나치게 캐주얼하거나 딱딱하지 않음
- **3점**: 대체로 적절하나 일부 표현이 너무 구어체이거나 모호함
- **1점**: 구어체·은어·지나친 겸양 또는 단호한 단정이 인터뷰 맥락에 부적절

평가 시 확인할 항목:
- "그냥", "뭐", "어", "좀" 등 지나친 구어체 사용 여부
- "잘 모르겠는데요", "틀릴 수도 있지만" 등 자신감 저하 표현
- 지나친 단정 ("항상 이게 맞다", "무조건") — 유연성 부족 표현
- 면접관에 대한 적절한 존칭과 격식

### 3단계: 출력 포맷

```
## 답변 평가 리포트

**질문**: [질문 내용]

---

### 종합 점수

| 기준 | 점수 | 한 줄 평가 |
|------|------|-----------|
| 의도 부합도 | X / 5 | [요약] |
| 깊이 | X / 5 | [요약] |
| 톤앤매너 | X / 5 | [요약] |
| **총점** | **X / 15** | |

---

### 상세 피드백

#### 의도 부합도 (X/5)
[구체적 피드백 — 잘된 점과 부족한 점]

#### 깊이 (X/5)
[구체적 피드백 — 무엇을 더 추가해야 하는지]

#### 톤앤매너 (X/5)
[구체적 피드백 — 고쳐야 할 표현 인용]

---

### 개선 포인트 요약
1. [가장 중요한 개선 사항]
2. [두 번째 개선 사항]
3. [세 번째 개선 사항]
```

사용자가 원하면 개선된 답변 예시를 추가로 작성한다.
