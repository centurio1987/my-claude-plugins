# 포지션 프리셋

각 포지션별 핵심 개념 리스트. 질문 생성 시 이 리스트를 기반으로 커버리지를 확인한다.

---

## node.js engineer

### 핵심 개념
- **이벤트 루프**: 단일 스레드 비동기 처리 모델, 콜 스택/태스크 큐/마이크로태스크 큐
- **async/await & Promise**: 비동기 흐름 제어, 에러 처리, Promise.all/race/allSettled
- **스트림(Stream)**: Readable/Writable/Transform 스트림, 백프레셔(backpressure)
- **모듈 시스템**: CommonJS vs ESM, 동적 임포트
- **Express/Fastify**: 미들웨어 구조, 라우팅, 에러 핸들링, 성능 차이
- **TypeScript**: 타입 시스템, 제네릭, 데코레이터, tsconfig
- **테스팅**: Jest/Vitest, 유닛·통합·E2E 테스트, 모킹
- **성능 최적화**: 프로파일링, 메모리 누수, 클러스터링, Worker Threads
- **보안**: JWT, CORS, rate limiting, helmet, SQL injection 방어
- **패키지 관리**: npm/pnpm/yarn, 의존성 관리, lockfile

### 경력별 강조 포인트
- **주니어(1~3년)**: 이벤트 루프 이해, 기본 API 설계, 테스트 작성
- **미드(3~5년)**: 성능 튜닝, 아키텍처 패턴, 시스템 설계 기초
- **시니어(5년+)**: 대규모 시스템 설계, 기술 선택 근거, 팀 리딩

## product engineer

### 핵심 개념
- **시스템 설계**: 고가용성, 확장성, 데이터 모델링, API 설계
- **풀스택**: 프론트(React/Next.js) + 백(Node/Go/Python) 통합
- **API 설계**: REST vs GraphQL vs gRPC, 버전 관리, 문서화
- **데이터베이스**: RDBMS vs NoSQL 선택, 인덱싱, 쿼리 최적화, 마이그레이션
- **코드 품질**: 리팩토링, 테스트 커버리지, 코드 리뷰 문화
- **성능**: 웹 성능(Core Web Vitals), 캐싱 전략, CDN
- **UX 협업**: 디자인 시스템, 접근성, 피그마↔코드 워크플로우
- **기능 출시**: 피처 플래그, A/B 테스트, 점진적 롤아웃
- **기술 부채**: 우선순위 판단, 리팩토링 시점 결정

### 경력별 강조 포인트
- **주니어**: 기본 CRUD, API 연동, UI 구현
- **미드**: 시스템 설계, 성능 최적화, 프로덕트 감각
- **시니어**: 아키텍처 의사결정, 기술↔비즈니스 연결, 자율적 실행

## cto

### 핵심 개념
- **기술 전략**: 기술 로드맵 수립, 빌드 vs 바이 결정, 벤더 선택
- **팀 빌딩**: 채용 기준, 엔지니어 성장 체계, 조직 구조(팀 토폴로지)
- **아키텍처**: 마이크로서비스 vs 모노리스 트레이드오프, 기술 스택 통합
- **기술 부채**: 부채 측정, 리팩토링 투자 ROI, 레거시 시스템 전략
- **엔지니어링 문화**: 심리적 안전감, 장애 대응 문화(blameless), OKR
- **비용 관리**: 클라우드 비용 최적화, 엔지니어링 생산성 지표
- **보안/컴플라이언스**: 정보보호 거버넌스, GDPR/개인정보보호법
- **비즈니스 연계**: 기술 투자를 비즈니스 임팩트로 설명하는 능력
- **위기 관리**: 대형 장애 대응, 커뮤니케이션, 사후 분석

### 경력별 강조 포인트
- **스타트업 CTO**: 빠른 기술 선택, 소규모 팀 리딩, 직접 코딩
- **스케일업 CTO**: 조직 확장, 프로세스 구축, 채용/성장
- **대기업 CTO**: 거버넌스, 플랫폼 전략, 이해관계자 관리

## product owner

### 핵심 개념
- **프로덕트 비전**: 비전 수립, 로드맵 커뮤니케이션, 스테이크홀더 정렬
- **백로그 관리**: 우선순위 프레임워크(RICE/ICE/MoSCoW), 스프린트 계획
- **사용자 리서치**: 인터뷰, 설문, 사용성 테스트, 페르소나
- **데이터 기반 의사결정**: 지표 설정(KPI/OKR), 코호트 분석, A/B 테스트 해석
- **스테이크홀더 관리**: 경영진 보고, 개발팀·디자인팀 협업, 갈등 조정
- **Agile/Scrum**: 스프린트 리뷰/회고, 데일리 스탠드업, 번다운 차트
- **GTM(Go-to-Market)**: 출시 계획, 마케팅 협업, 출시 후 모니터링
- **경쟁사 분석**: 시장 조사, 포지셔닝, 차별화 전략
- **기술 이해**: 엔지니어와 소통할 수 있는 기술적 기초 지식

### 경력별 강조 포인트
- **주니어 PO**: 백로그 작성, 스프린트 운영, 사용자 스토리
- **미드 PO**: 지표 기반 의사결정, 로드맵 수립
- **시니어 PO**: 프로덕트 전략, 조직 영향력, P&L 이해
