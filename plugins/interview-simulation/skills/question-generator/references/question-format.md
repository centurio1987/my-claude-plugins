# 질문지 출력 포맷

## 파일명 규칙

```
interview-[포지션]-[YYYYMMDD].md
예: interview-nodejs-engineer-20260603.md
```

---

## 질문지 헤더

```markdown
# 면접 질문지 — [포지션명]

- 생성일: YYYY-MM-DD
- 경력: [연차]
- 포지션 디테일: [회사 규모, 도메인, 특이사항]
- 총 문항: N개

---
```

---

## 개별 질문 포맷

```markdown
## Q[번호]. [질문 내용]

| 항목 | 내용 |
|------|------|
| 중요도 | ⭐⭐⭐ (High) |
| 난이도 | Hard |
| 의도 | [이 질문으로 평가하는 역량] |
| 키워드 | `키워드1`, `키워드2`, `키워드3` |

---
```

---

## 중요도 기준

| 레이블 | 의미 |
|--------|------|
| ⭐ (Low) | 보조적인 내용, 나오면 좋은 수준 |
| ⭐⭐ (Medium) | 해당 포지션에서 빈출되는 질문 |
| ⭐⭐⭐ (High) | 반드시 준비해야 하는 핵심 질문 |

---

## 난이도 기준

| 레이블 | 의미 |
|--------|------|
| Easy | 기본 개념 확인, 주니어도 답할 수 있는 수준 |
| Medium | 실무 경험이 필요한 깊이의 질문 |
| Hard | 설계 판단력·트레이드오프 이해를 요구하는 질문 |

---

## 질문지 전체 예시

```markdown
# 면접 질문지 — Node.js Engineer

- 생성일: 2026-06-03
- 경력: 4년
- 포지션 디테일: B2B SaaS, 시리즈 B, 백엔드 중심
- 총 문항: 12개

---

## Q1. Node.js의 이벤트 루프가 어떻게 동작하는지 설명해 주세요.

| 항목 | 내용 |
|------|------|
| 중요도 | ⭐⭐⭐ (High) |
| 난이도 | Medium |
| 의도 | Node.js 비동기 처리 원리 이해도 및 성능 문제 진단 능력 평가 |
| 키워드 | `이벤트 루프`, `콜 스택`, `태스크 큐`, `마이크로태스크`, `libuv` |

---

## Q2. Promise.all과 Promise.allSettled의 차이를 설명하고, 각각 어떤 상황에 사용하시나요?

| 항목 | 내용 |
|------|------|
| 중요도 | ⭐⭐ (Medium) |
| 난이도 | Easy |
| 의도 | 비동기 병렬 처리 패턴 활용 능력 평가 |
| 키워드 | `Promise.all`, `Promise.allSettled`, `에러 전파`, `부분 실패 처리` |

---
```
