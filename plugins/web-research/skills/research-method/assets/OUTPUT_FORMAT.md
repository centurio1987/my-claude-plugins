# 산출물 형식 (최종 리서치 문서)

워커 findings와 오케스트레이터 최종본이 **공용**으로 쓰는 한국어 문서 형식이다. 최종 파일은
`<outputDir>/<YYYY-MM-DD>-<slug>.md`로 저장한다(날짜는 KST 기준).

```markdown
---
title: <사람이 읽는 제목>
topic: <주제>
collected: <조회일 YYYY-MM-DD (KST)>
sources_count: <전역 신뢰 소스 개수>
---

## 개요
<2~4문장 요약: 무엇에 대한 자료이고 왜 모았는가. 핵심 결론 한 줄 포함>

## <소주제 1>
<충실한 정리. 키워드가 아니라 실제 내용(정의·수치·동작·예시·짧은 인용).
 핵심 사실 문장 끝에 (S1)(S2) 식으로 소스 번호 표기>

## <소주제 2>
...

## Open questions (부족·미결 자료)
- <아직 확인 못 한 것 / 충돌해서 미결인 것 / 페이월·차단으로 막힌 것>

## 주장 ↔ 소스 매핑
- <핵심 주장 A>: S1, S3
- <핵심 주장 B>: S2
- <충돌 항목>: S4(주장 X) vs S5(주장 Y) — 양쪽 기록, 신뢰도 근거 메모

## 소스 목록
- S1. <제목> — <URL> (게시일: YYYY-MM-DD, 조회일: YYYY-MM-DD)
- S2. ...
```

## 규칙

- **소스 번호 전역 유일**: 최종 문서의 `S1…SN`은 전역 고유번호다. 워커별 임시번호는 오케스트레이터가
  병합 시 이 전역번호로 재매핑한다(본문 인라인 `(S?)` 표기도 함께 치환해 링크가 깨지지 않게).
- **`sources_count`**: 중복 제거 후 남은 전역 신뢰 소스 개수. 전체 문서 기준 **최소 5개**를 지향한다.
- **게시일·조회일 필수**: 알 수 없으면 "게시일: 미상"으로 명시(공란 금지).
- **충돌 정보**: 주장↔소스 매핑과 본문에 양쪽 다 남긴다. 어느 쪽이 더 신뢰되는지 근거를 적되 임의 단정 금지.
- **언어**: 한국어. 단 수치·버전·날짜·고유명사·짧은 인용은 원문 그대로.
