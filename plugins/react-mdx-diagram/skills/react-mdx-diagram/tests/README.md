## react-mdx-diagram · Playwright Test Harness

Run with: `cd tests && npm i && npm test` (which executes `node render.mjs && playwright test`).

The harness first compiles each skin's fixture (`assets/samples/reference.<skin>.mdx`) via `@mdx-js/mdx` `evaluate()`, renders the React component to static HTML with `react-dom/server` `renderToStaticMarkup()`, injects an inline Helvetica Neue / JetBrains Mono font stack, and writes the result to `tests/.out/<skin>.html`. Playwright then loads each file via `file://` URL and asserts the frozen CONTRACT §1–§5 invariants: body/panel background matches `surface.canvas` per skin (`#0B0B0B` candy-os, `#ECEBE7` swiss, `#F4F3F0` blueprint); all 8 palette hexes appear in the rendered markup; all 5 taxonomy node types are present and their fill colors are correct (person purple, external orange, container green, database blue, queue yellow); C4 shows 3 nested levels whose expression method matches each skin (blueprint decreasing border widths 3px→2px→1.4px, swiss increasing left indent, candy-os rgba opacity ramp); at least one element carries a `Helvetica Neue` font-family and one carries `JetBrains Mono`; blueprint nodes have a `2.5px solid #111` border while candy-os and swiss do not; and a final cross-skin assertion confirms the three HTML canvases are distinct. Chromium is expected to be pre-cached at `~/Library/Caches/ms-playwright` — do **not** run `npx playwright install`.
