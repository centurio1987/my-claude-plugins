# animate 스킬 (애니메이션 오케스트레이터)

프론트엔드 결과물을 받아 **포착(detection) → 게이트웨이(score) → 승인 → 적용(application)**의
순서로 진행한다. 핵심 최적화는 **포착·적용을 서브에이전트로 격리**하는 것이다 — 코드 본문 덤프와
파일 수정이 메인 대화를 오염시키지 않도록, 워커는 정제된 결과만 반환하고 이 스킬은 그걸 조율한다.

> **이 스킬(메인 스레드)은 긴 가이드를 읽지 않는다.** 상세 규칙(`ANIMATION_GUIDE`·`REFERENCE_SPECS`)은
> 워커에게만 로드된다. 메인은 짧은 `OUTPUT_CONTRACT`와 `SCORE_CHECKLIST`의 **임계값·무조건 제외·
> 후보 상한·승인 출력 형식**만 참조한다(컨텍스트 절감). 이 항목들은 체크리스트에 메인 단독으로
> 이해 가능하게 적혀 있다.

## 입력 / 출력 계약

- **입력**: 프론트엔드 코드 경로(파일/디렉터리) 또는 현재 변경분. (선택) 스크린샷, (선택) 디자인 시스템/모션 토큰.
- **출력**: 승인 영역에 적용된 모션 코드 변경 + 적용 결과 요약(영역·패턴·변경 파일·롤백 방법).
- **불변 원칙**: 추측 변경 금지. 과잉 모션 금지. 무조건 제외 조건은 점수와 무관하게 제외.
  검증 실패 시 자동 롤백. 사용자 승인 없이 디자인 시스템 파일을 수정하지 않는다.

## 동작 순서

### 1. 사전 검사 / 실패 모드

대상이 프론트엔드인지 확인한다. 다음이면 **적용을 멈추고 그 사실을 보고**한다(추측 변경 금지):

- 프론트엔드 경로를 못 찾음 / 백엔드 전용 repo / 읽기 전용 환경 / monorepo에서 대상 패키지 불명확.

### 2. 포착 디스패치 (`animation-scout`)

대상 범위를 정해 `animation-scout`를 디스패치한다.

- **탐색 스케일 규칙**: 최대 탐색 파일 수·디렉터리 깊이 상한을 둔다. 초과 시 **디렉터리/컴포넌트
  단위로 슬라이싱**해 분할 디스패치한다(전체 코드베이스 일괄 투입 금지 — 토큰 오버플로우 방지).
- scout는 후보마다 **1순위 패턴 + 2순위 대안 + 항목별 점수/총점 + 무조건-제외 플래그**를 반환한다.

### 3. 타당성 게이트웨이 (One-pass, scout 재호출 없음)

scout가 반환한 후보별 점수를 `SCORE_CHECKLIST`의 **임계값**과 비교해 판정한다:

- 1순위 총점 `≥ 임계` → **1순위 채택**.
- 1순위 미달이지만 2순위 `≥ 임계` → **2순위로 교체**(약한 강도).
- 둘 다 미달 **또는 무조건-제외 플래그** → **제외**(스킵 로그에 사유 남김).
- 별도 재평가 라운드는 없다(scout를 다시 부르지 않는다).

### 4. 사용자 승인 게이트 (기본 ON)

통과 후보를 **체크리스트식**으로 제시한다 — 후보별 영역·채택 패턴·강도·총점·예상 변경 파일.
상위 N개(기본 ≤3)를 제안하고, **후보별 선택/제외/강도 조정**을 받는다.

- `--auto`(또는 사용자가 자동 적용을 명시)면 이 단계를 생략하되, **변경 범위 상한은 그대로 강제**한다.

### 5. 적용 디스패치 (`animation-applier`)

승인된 영역 목록을 `animation-applier`에 전달한다. 적용자는 스택 감지 → 의존성 정책(기존 의존성만,
없으면 CSS 폴백) → 구현(reduced-motion·transform/opacity) → **린트/빌드 검증 + 실패 시 자동 롤백**을
수행하고 결과를 반환한다.

### 6. 보고

적용 결과를 요약한다: 적용 영역·패턴·근거·변경 파일·토큰 위치·접근성 처리·검증 결과·**롤백 방법**,
그리고 게이트웨이에서 제외된 후보와 사유.

## 변경 범위 제한 (기본값)

1회 실행당 **후보 ≤3개, 수정 파일 ≤5개, 신규 공유 토큰 파일 ≤1개**. 디자인 시스템/토큰 파일은
사용자 승인 없이는 수정·생성하지 않는다. `--auto`에서도 이 상한은 강제된다.

## 금지·필수

- 출처/근거 없는 모션, 과잉 모션, 추측 기반 변경 금지.
- 무조건 제외 조건 해당 영역은 점수와 무관하게 제외한다.
- 검증 실패 시 코드베이스를 원상 복구(자동 롤백)한 뒤 보고한다.
