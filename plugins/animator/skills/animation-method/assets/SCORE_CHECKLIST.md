# SCORE_CHECKLIST — 적용 타당성 점수 체크리스트 (포착자·게이트웨이 공용 단일 출처)

포착자(`animation-scout`)는 후보를 이 기준으로 점수화하고, 게이트웨이(`animate`)는 이 임계값으로
통과/제외를 판정한다. **점수 산정과 통과 판정의 단일 출처**다 — scout 점수 기준 ≡ 게이트웨이 임계.

> 메인 오케스트레이터가 워커 가이드를 읽지 않아도 판단할 수 있도록, **임계값·무조건 제외·후보 상한·
> 승인 출력 형식**은 이 문서만으로 완결적으로 이해 가능하다.

## 점수 구성 (양의 항목 합 100 − 감점, 0~100으로 clamp)

| # | 기준 | 배점 | 무엇을 보는가 |
|---|------|------|----------------|
| 1 | **의미적 적합성** | 0–30 | 모션이 의미·위계·인과(원인→결과)를 전달/강화하는가 |
| 2 | **모션 어포던스** | 0–25 | 상태변화/인터랙션(hover·expand·load·진입·전환)이 모션을 자연히 부르는가 |
| 3 | **주의 가치** | 0–20 | 첫인상·CTA·핵심 정보 등 사용자 주의를 쓸 만한 영역인가 |
| 4 | **성능 안전성** | 0–15 | transform/opacity만으로 구현 가능한가(레이아웃 thrash 없음) |
| 5 | **접근성** | 0–10 | `prefers-reduced-motion` 대응 가능, 과함/어지럼(vestibular) 위험 낮음 |
| − | **과잉 위험(감점)** | 0~ −20 | 이미 모션 과다/중복/산만/브랜드 톤 부적합 |

**총점 = (1+2+3+4+5) − 과잉위험**, 0~100으로 clamp.

## 임계값 / 게이트웨이 판정

- **임계값(기본): 60/100.**
- 게이트웨이는 **1순위 → 2순위 → 제외** 순으로 판정한다(scout 재호출 없음):
  - 1순위 총점 ≥ 60 → 1순위 채택.
  - 1순위 < 60 이고 2순위 ≥ 60 → 2순위로 교체.
  - 둘 다 < 60, **또는 무조건 제외 조건 해당** → 제외.

## 무조건 제외 조건 (점수와 무관하게 제외)

- `prefers-reduced-motion` 대응이 불가능한 모션.
- 해당 영역에 이미 충분한 모션이 존재(중복·충돌 위험).
- 접근성 속성(focus 이동·aria-expanded·focus trap)이나 기존 이벤트 핸들러를 깨뜨려야만 적용 가능.
- 정적 분석으로 기존 모션을 확인할 수 없어 충돌 위험이 큰 영역(scout가 "확인 불가"로 표기).

## 정량 상한 (Hard Ceiling — 적용 시 반드시 준수)

근거: Nielsen Norman Group, Material 3, web.dev (상세·출처는 `REFERENCE_SPECS.md`).

- **단일 애니메이션 duration**: **100–500ms** 범위. **500ms 이상 금지**(지연감). 단순 피드백 ~100ms,
  hover 150–200ms, 진입·모달 200–300ms. 진입 > 퇴장(비대칭).
- **응답 시작**: 사용자 액션 후 **0.1초(100ms) 이내** 모션이 시작해야 직접 조작감 유지.
- **stagger 적용 요소 최대 개수**: **최대 10–15개**, 항목 간 delay 30–80ms, 총 시퀀스 **≤ ~800ms**.
  초과분/무한 스크롤은 IntersectionObserver로 **뷰포트 진입분만**. 초기 전체 로드 일괄 stagger 금지.
- **동시 모션 / 뷰포트 점유**: 한 화면에서 주의를 끄는 동시 모션은 소수로. 화면 대부분을 덮는 동시
  모션 지양(인지 과부하). 간접 트리거 콘텐츠는 느리고 위치 변화 적게.
- **금지 속성**: layout-triggering 속성(top/left/right/bottom/width/height/margin/padding) 애니메이트
  금지 — `transform`(translate/scale/rotate)·`opacity`만(컴포지터 60fps). `box-shadow`/`background`는 절제.
- **`will-change`**: 남용 금지(상시 부착 금지, 모션 직전 부착 → 종료 후 제거).
- **무한 반복 모션**: 주의를 끄는 무한 루프(자동재생 패럴랙스/캐러셀)는 과잉위험 감점 + 정지 컨트롤
  필수 + reduced-motion에서 정지. 로더의 무한 회전은 예외(컴포지터 속성 한정).

## 점수 구간 예시 (재현성 — "숫자로 포장된 주관" 방지)

각 기준을 매길 때 아래 예시에 비춰 근거를 단다.

**1. 의미적 적합성 (0–30)**
- 27–30: 진입/전환이 화면 구조·인과를 직접 설명(모달 열림, 라우트 전환, 단계 진행).
- 14–20: 위계·강조를 보조(hero 텍스트 순차 등장).
- 0–8: 장식적일 뿐 의미 전달 없음(배경 도형이 의미 없이 떠다님).

**2. 모션 어포던스 (0–25)**
- 22–25: 명확한 상태변화·인터랙션(hover, 아코디언 expand, 로딩→완료, 토글).
- 12–18: 진입형(스크롤 리빌)처럼 트리거는 있으나 상호작용은 약함.
- 0–6: 정적 텍스트 블록 등 상태변화 없는 영역.

**3. 주의 가치 (0–20)**
- 17–20: 첫인상 hero, 주요 CTA, 결과/성공 피드백.
- 9–14: 보조 카드·섹션 제목.
- 0–5: 푸터 약관, 반복되는 보조 메타데이터.

**4. 성능 안전성 (0–15)**
- 13–15: transform/opacity만으로 완결(translate/scale/fade).
- 6–10: 일부 레이아웃 영향 있으나 회피 가능(height→max-height/grid-rows 기법 필요).
- 0–4: 본질적으로 레이아웃 리플로우 유발(목록 재정렬, 텍스트 리플로우).

**5. 접근성 (0–10)**
- 9–10: reduced-motion 폴백 자명, content shift·focus 영향 없음.
- 4–7: 폴백 가능하나 focus/aria 동기화에 주의 필요(모달·드로어·아코디언).
- 0–3: vestibular 위험 큼(대형 패럴랙스, 급격한 스케일/회전).

**과잉 위험 (−0 ~ −20, 감점)**
- −15~−20: 영역에 이미 모션 과다 / 자동재생 무한 루프 다수 / 브랜드 톤(업무툴인데 과한 모션) 충돌.
- −5~−10: 인접 영역과 중복되는 진입 모션, 약한 산만함.
- 0: 과잉 징후 없음.

## hard exclude 예시

- 데이터 대시보드의 조밀한 테이블 행마다 진입 stagger → vestibular·산만 → 제외.
- 이미 Framer Motion 진입 모션이 있는 카드에 또 CSS 진입 추가 → 중복 → 제외.
- 클릭 핸들러가 달린 요소를 모션 래퍼로 감싸 이벤트가 끊김 → 핸들러 파손 → 제외(또는 핸들러 보존 재설계).

## 모션 가치가 높은 영역 (NN/G 4대 기능 범주 — 의미적 적합성·어포던스 가점 근거)

1. **피드백/마이크로인터랙션** — 액션이 인식됐음을 확인(change blindness 극복).
2. **상태 변화 전달** — 모드/뷰 전환, 사용자가 트리거하지 않은 변화(로딩 등).
3. **공간 내비게이션/wayfinding** — 위계 내 위치(zoom=상하 레벨, slide=전후 이동).
4. **인터랙션 signifier** — 요소 동작 방식 시연(모션 방향이 허용 동작 암시).

장식적·시간 때우기용 모션, 반복 노출로 짜증을 유발하는 모션, 동시 다발 모션, 콘텐츠 표시를 지연시키는
모션은 **과잉위험 감점** 대상이다.
