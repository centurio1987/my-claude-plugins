# OUTPUT_CONTRACT — 출력 계약

`animation-scout`(포착자)와 `animation-applier`(적용자)가 반환하는 텍스트 형식을 고정한다. 메인
오케스트레이터(`animate`)는 이 형식만 보고 게이트웨이·승인·보고를 처리한다(긴 가이드를 읽지 않음).

---

## 1. scout findings 형식 (포착자 → 게이트웨이)

먼저 **기존 모션 인벤토리**와 **스택/버전**을 짧게 요약한 뒤, 후보 목록을 반환한다.

```
## 스택/버전
- 프레임워크: <React 18 / Vue 3 / ...>
- 모션 라이브러리: <Framer Motion 11 / 없음 / Tailwind animate / GSAP ...>
- 스타일링: <CSS Modules / Tailwind v3 / styled-components / ...>

## 기존 모션 인벤토리
- <파일:요소> — <이미 적용된 모션 요약>
- 확인 불가/충돌 위험: <CSS-in-JS·동적 className 등 정적 분석 한계 영역>

## 후보 (점수 높은 순)
### C1. <영역 이름> — <파일경로:요소/컴포넌트>
- 의미적 역할: <hero / CTA / 리스트 / 모달 / 로더 ...>
- 1순위: <패턴> (강도: <subtle|standard|expressive>)
- 2순위(대안): <패턴> (강도: <더 약함>)
- 점수: 의미적합 __/__, 주의가치 __/__, 어포던스 __/__, 성능 __/__, 접근성 __/__, 과잉위험(감점) -__
- 총점: __/100
- 무조건-제외: <없음 | 사유(reduced-motion 불가 / 기존 모션 충분 / 접근성·핸들러 파손)>
- 근거: <왜 이 패턴/점수인지 — 체크리스트 구간 예시에 비춰 1~2줄>
- 입력 기반: <코드 기반 | 스크린샷 보강>
```

- 후보가 없으면 "적용 가치 있는 후보 없음"과 사유를 명시한다.
- 스크린샷이 없으면 모든 후보에 `입력 기반: 코드 기반`을 적고, 시각 확인이 필요한 항목은 그 사실을 표기.

---

## 2. applier 보고 형식 (적용자 → 오케스트레이터)

```
## 적용 결과
### A1. <영역 이름> — <파일경로>
- 적용 패턴/강도: <fade-up / stagger ...>
- 구현 방식: <Framer Motion variants | CSS keyframes | Tailwind transition ...>
- 변경 파일: <경로 목록>
- 토큰 정의 위치: <기존 CSS var 재사용 / theme / 신규 파일(승인됨) ...>
- 기존 스타일 충돌: <없음 | 처리 내용>
- 접근성: prefers-reduced-motion <대응 방식>, focus/aria <보존 처리>
- 검증: 린트 <pass/fail>, 빌드·타입체크 <pass/fail/실행불가>
- 롤백 방법: <git restore <files> | 제거할 신규 파일 | 역-Edit 요약>

## 제외/스킵
- <영역> — <사유(임계 미달 / 무조건 제외 / 미승인)>

## 검증 실패 시
- 자동 롤백 수행 여부와 원복된 파일, 실패 사유.
```

- 검증 실패 시: 보고에 그치지 말고 **자동 롤백**을 수행했음을 명시하고, 코드베이스가 원상 복구됐음을 확인.
- 검증 명령이 없는 환경이면 그 사실 + 수동 검증 요청을 명시한다.
