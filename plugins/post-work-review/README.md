# post-work-review

PLAN.md / BACKLOG.md 기반 작업을 끝낸 뒤, **그 최종 산출물을 분석해 재사용 가능한 패턴을 찾아주는**
사후 작업 검토 키트입니다. 훅 + 스킬 + 에이전트 세 가지를 하나의 플러그인으로 묶었습니다.

## 무엇을 하나

작업이 끝난 결과물에서 아래 세 종류의 후보를 식별해 **제안하고, 고른 것만 생성**합니다.

- **(A) 반복·식별된 작업 → 스킬 후보**
- **(B) 하위 디렉토리 `CLAUDE.md` → 프로젝트 스코프 룰 후보**
- **(C) 병렬화 가능한 작업 → 에이전트 후보**

> 이 플러그인은 "후보를 찾아 제안하는 도구"입니다. 실제 생성은 항상 사용자가 고른 항목만,
> 초안 형태로 수행하며 **원본 파일을 자동 삭제·이동하지 않습니다.**

## 구성 요소

| 종류 | 이름 | 역할 |
|---|---|---|
| 훅 (Stop) | `hooks/detect-completion.sh` | PLAN.md/BACKLOG.md 체크리스트가 **모두 완료**되면, `/post-work-review` 를 권하는 알림을 1회 표시 |
| 스킬 | `post-work-review` | 산출물 분석 → 후보 리포트 → 사용자 선택 → 초안 생성까지 오케스트레이션 |
| 에이전트 | `work-pattern-analyzer` | 산출물을 읽기 전용으로 스캔해 세 카테고리 후보를 고정 스키마로 반환 |

## 훅 트리거 조건

- `PLAN.md` 또는 `BACKLOG.md` 에 GFM 체크박스가 있고, **체크박스를 가진 파일이 모두 완료**
  (미완료 `- [ ]` 0개, 완료 `- [x]` 1개 이상)일 때만 1회 알림.
- fenced code block(``` ``` ```)과 HTML 주석(`<!-- -->`) 안의 체크박스는 무시합니다.
- 같은 상태(파일 내용 + git HEAD/diff)에서는 다시 알리지 않습니다(중복 방지).
- 상태 파일은 프로젝트의 `.claude/.post-work-review-state` 에 기록되며, git 레포면 `.git/info/exclude` 에
  자동 등록되어 레포를 더럽히지 않습니다. 어떤 오류에서도 작업 흐름을 깨지 않도록 **항상 정상 종료**합니다.

## 사용법

설치 후, 작업을 마치면 다음 중 하나로 검토를 시작합니다.

- Stop 훅 알림을 보면 `/post-work-review` 실행
- 또는 직접: "사후 작업 검토해줘" / `/post-work-review`

생성물은 **현재 프로젝트의 `.claude/`** 아래에 만들어집니다
(스킬→`.claude/skills/`, 에이전트→`.claude/agents/`, 룰→`.claude/CLAUDE.md`).

## 테스트

```bash
bash hooks/test-detect.sh
```

체크박스 변형, 코드블록/주석 제외, 분할 완료, 중복 방지 등 케이스를 검증합니다.
