# Process Order (문서 기반 지시 처리)

`<project-root>/ORDER.md`는 인터랙티브 대화 대신 **문서로 지시를 받는 창구**다. 이 스킬은 그 지시를 수행하고,
완료된 지시를 **편집 금지(봉인) 구획**으로 옮겨 `committed` 됐음을 표현한다.

## 핵심 규칙

- 사용자가 편집하는 영역은 `## 신규 지시` **하나뿐**이다.
- 완료된 지시는 `## 처리 완료 (COMMITTED)` 영역에 **주석 쌍으로 구획된 봉인 블록**으로 기록한다.
- **봉인 블록은 절대 수정·삭제하지 않는다.** 지시의 철회·정정은 이전 내용을 고치는 게 아니라,
  `## 신규 지시`에 **새 revert 지시**(`reverts=ORD-NNN`)를 추가해 되돌린다(git revert 의미론).

## 절차

### 1. 재개 점검 (원자성)

먼저 ORDER.md 전체를 읽고, `status=processing` 마커가 달린 **미완 블록**이 있는지 확인한다.
- 있으면: 그 지시가 실제로 이미 수행됐는지 산출물·파일 상태로 판단한다.
  - 이미 반영됐으면 → 추가 작업 없이 곧장 **봉인(4단계)**으로 전환한다.
  - 아직이면 → 이어서 수행한다.
- 이 점검으로 세션 중단 시 **같은 지시의 중복 수행**을 방지한다.

### 2. 신규 지시 읽기

`## 신규 지시` 영역의 각 지시 블록을 읽는다. `## 처리 완료 (COMMITTED)` 영역은 **읽기만** 하고 수정하지 않는다.
봉인 영역에서 짝이 맞지 않는 마커 등 사람이 변형한 흔적이 보이면, 수행을 멈추고 그 사실을 경고로 보고한다.

### 3. 수행 (수행 중 상태 먼저 기록)

한 지시를 처리하기 직전에:
1. 다음 순번 `ORD-NNN`을 부여하고, 해당 지시 블록에 `status=processing` 마커를 붙여 **"수행 중"임을 먼저 기록**한다(세션 중단 대비).
2. 지시를 수행한다.
   - 일반 지시: 요청된 작업을 수행한다.
   - **revert 지시**(`reverts=ORD-MMM`): 원본 ORD-MMM이 만든 변경을 되돌리는 작업을 한다. 원본 봉인 블록은 건드리지 않는다.

### 4. 봉인 (committed)

수행이 끝나면 그 지시를 `## 신규 지시`에서 제거하고, `## 처리 완료 (COMMITTED)` 영역 끝에 봉인 블록으로 옮긴다:

```
<!-- ORDER:COMMITTED id=ORD-NNN status=done committed=YYYY-MM-DD -->
<!-- 봉인 구획: 편집 금지. 철회하려면 신규 지시에 reverts=ORD-NNN 항목을 추가하세요. -->
### ORD-NNN — {지시 요약}
- 지시: {원문}
- 결과: {수행 결과 요약 · 관련 파일/커밋}
<!-- /ORDER:COMMITTED id=ORD-NNN -->
```

- revert 지시는 `status=reverted reverts=ORD-MMM` 로 표기한다.
- `processing → done`(또는 `reverted`) 전이는 한 번에 마무리해, 봉인 안 된 미완 상태로 남기지 않는다.

### 5. 보고

처리한 지시 ID·결과와, 봉인 후 남은 신규 지시 유무를 한 줄로 보고한다.
