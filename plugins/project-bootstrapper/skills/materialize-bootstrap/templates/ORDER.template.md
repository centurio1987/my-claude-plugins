# ORDER — {프로젝트명}

> **문서 기반 지시 창구.** 인터랙티브 대화 대신 이 파일에 지시를 적으면, 이 프로젝트의 에이전트가
> 수행합니다. (운영 방식 자체를 바꾸는 일은 `<관리폴더>/PLAN.md`에서 다룹니다 — 여기는 상시 지시용.)
>
> **사용법**
> 1. 아래 `## 신규 지시` 영역에만 지시를 적습니다(유저가 편집하는 유일한 영역).
> 2. 에이전트가 `process-order` 스킬로 지시를 수행하고, 완료되면 그 지시를 아래
>    `## 처리 완료 (COMMITTED)` 로그로 옮겨 **편집 금지 구획**으로 봉인합니다.
> 3. 봉인된 블록(`<!-- ORDER:COMMITTED ... -->` … `<!-- /ORDER:COMMITTED ... -->` 사이)은
>    **수정·삭제하지 마세요.** 지시를 철회·정정하려면 이전 내용을 고치는 게 아니라,
>    `## 신규 지시`에 **새 revert 지시**(`reverts=ORD-NNN`)를 적습니다. (git revert 의미론)
